Highlight #1 Version Control

Version control is the process of keeping track of all versions of files. This is very important, especially for projects which have multiple people contributing so as to not have people constantly overwriting each others work throughout the process.

Highlight #2 VCS

A VCS, or Version Control System, is a system which manages the different versions of projects. It organizises the different versions in such a way that all of the peopleworking on the project are able to work on the most current version or have access to older versions if necessary.

Highlight #3 Local Repository

The local repository is a copy of your files stored locally before being sent to the remote repository

Highlight #4 Remote Repository

You push the local repository to this repository in order to store the data off site. This allows for redundancy as the files are stored in two places.

Highlight #5 Git

Git is a very popular source code management system for software development. It is a distributed revision control system with a focus on speed, data integrity, and support for distributed, non-typical workflows.
